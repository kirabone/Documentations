Versioning Strategy

chatXi follows a structured semantic-like versioning format: vX.Y.Z

  • X — Stage
    Represents the major project phase as described above.
    Example:
    • 0 → CLI-based system
    • 1 → GUI introduction
    • 2 → AI-driven participants
    • 3 → Autonomous social agents
    • 4 → Human-based AI cloning
  • Y — Feature Progress
    Indicates the current development milestone inside a stage, progressing in small, chronological steps that mirror how a real chat application evolves.

    Example (Stage 0):
    • 0 → Authentication introduced
    • 1 → One-to-one messaging
    • 2 → Multi-user rooms
    • 3 → Presence & status handling
    • 4 → Persistence & logging
  • Z — Iteration
    Incremented on every small but meaningful code change, refactor, or experimental adjustment — often updated daily or whenever a piece of logic evolves.

This approach allows the project to move forward in very small, understandable steps while keeping changes traceable and reversible.

Development Philosophy

chatXi is built by beginners learning real systems. That means pauses, detours, and temporary abandonment are expected — and accepted.

At times, development may stop mid-stage if more skills are required. In such cases, the project may be put on hold, redirected to smaller or easier experiments, or temporarily simplified.

However, the journey itself is never abandoned. The project is designed to be returned to later with stronger skills, better understanding, and clearer intent — without treating earlier attempts as failure.
